# DB Chat — 自然语言数据库查询系统

## 背景

在日常业务运营中，产品经理和运营人员经常需要查看埋点数据、验证业务指标、排查数据异常。传统方式下，这些工作依赖手动编写 SQL 在数据库客户端中执行——对于非技术背景的同事而言，这一过程存在明显的痛点：

- SQL 语法门槛高，编写耗时且容易出错
- 不熟悉表结构和字段关系，需要反复询问开发同事
- 查询结果为原始数据行，缺乏直观的统计与可视化
- 缺少执行记录和上下文连续性，同类查询需反复编写

基于上述问题，我们开发了 DB Chat，旨在将数据库查询的门槛降低到"用自然语言描述需求"这一水平。

## 系统概述

DB Chat 是一个对话式数据库查询系统。用户通过自然语言描述数据需求，系统自动完成表结构分析、SQL 生成、执行确认、结果可视化的完整链路。

核心设计原则：

- **安全优先**：所有 SQL 均需用户确认后方可执行，支持只读/读写权限模式
- **准确可控**：AI 生成 SQL 前必须先查询真实表结构，禁止凭空编造数据
- **交互友好**：复杂查询通过交互控件引导参数选择，而非纯文本问答

## 技术架构

```
┌───────────────────────────────────────────────┐
│                 前端 (React 19)                │
│  TanStack Router · Tailwind CSS · Recharts    │
├───────────────────────────────────────────────┤
│            服务端 (TanStack Start)             │
│  Server Functions · SSE Streaming             │
├───────────────────────────────────────────────┤
│              AI Agent 层                       │
│  Tool Calling · Multi-step Reasoning          │
├───────────────────────────────────────────────┤
│             数据库连接层 (mysql2)              │
│  连接管理 · SQL 安全校验 · Schema 缓存        │
└───────────────────────────────────────────────┘
```

**前端技术栈**：React 19 + TanStack Router + Tailwind CSS 4 + Recharts

**服务端**：基于 TanStack Start 全栈框架，支持 Server Functions 和 SSE 流式输出

**AI 引擎**：接入 DeepSeek、通义千问（百炼）等多模型，支持 Thinking Mode 深度推理

**数据存储**：会话数据使用 IndexedDB（Dexie）本地持久化，数据库连接信息加密存储

**部署方式**：Docker 容器化构建，通过云效流水线自动部署至内网服务器

## 核心能力

### 1. 自然语言转 SQL

用户输入业务问题，系统自动分析数据库结构、生成准确的 SQL 查询。支持单表简单查询到多表 JOIN、子查询、窗口函数等复杂场景。

### 2. 智能查询规划

面对复杂问题（多表关联、多步依赖查询），系统先制定查询计划、自动获取涉及表的字段信息，再逐步执行，确保生成的 SQL 准确可靠。

### 3. 交互式参数筛选

当查询存在歧义参数（时间范围、枚举值、聚合粒度等），系统自动呈现交互控件供用户选择，避免反复文字沟通。

### 4. SQL 安全防护

- 基于 AST 解析的多层安全校验，拦截危险操作（DROP、TRUNCATE 等）
- 只读/读写权限模式可配置
- 所有 SQL 需用户确认后方可执行
- 单会话 SQL 执行次数上限控制

### 5. 结果可视化

查询结果自动提供三种视图：

- **表格视图**：支持排序、统计汇总行
- **图表视图**：自动推荐图表类型（柱状图、折线图、饼图、面积图），可手动切换
- **摘要视图**：KPI 卡片展示或字段统计分析

支持一键导出 CSV 和复制 Markdown 表格。

### 6. 数据脱敏

对查询结果中的手机号、身份证号、银行卡号、邮箱等敏感信息自动脱敏处理，保障数据安全合规。

### 7. 幻觉检测

内置独立分类器，对 AI 回复进行实时校验，识别并拦截可能编造的数据结果，确保用户看到的数据均来自真实查询。

### 8. 会话管理

支持多会话并行，每个会话维护独立的上下文和执行记录。历史记录本地持久化，支持会话自动命名。

## 安全与隐私

### 网络安全

- 系统部署于公司内网，不对外暴露任何端口，数据流转全程在内网完成
- 仅需内网数据库可达，无外部数据泄露路径

### SQL 执行安全

- 所有 SQL 必须经用户手动确认后方可执行，AI 无法自主执行任何语句
- 基于 AST 语法树解析 + 正则双重校验，拦截 DROP、TRUNCATE、ALTER 等危险操作
- 支持只读 / 读写两种权限模式，按连接独立配置
- 单会话 SQL 执行次数设有上限（默认 20 次），防止异常循环

### 环境隔离

- 每个数据库连接需标注环境（dev / test / staging / prod）
- 生产环境连接在 UI 层有醒目标识，降低误操作风险

### 密钥与凭证加密

- 数据库密码与 API Key 均采用 AES-256 加密存储，不以明文传输或持久化
- 服务端密钥解密仅在内存中瞬时完成，不落盘

### 数据隐私

- 所有会话记录存储于用户浏览器本地（IndexedDB），不上传至任何服务端
- 查询结果中的手机号、身份证号、银行卡号、邮箱等敏感字段自动脱敏
- 系统不留存用户查询历史，关闭浏览器即与服务端无关联

### AI 数据安全说明

系统通过标准 API 协议调用大模型推理服务，调用过程中用户的查询内容和数据库结构信息会通过 HTTPS 加密传输至模型供应商。以下为各供应商的数据政策：

**阿里云百炼（DashScope）**：

> "阿里云严格保护数据隐私，**绝不会将您的数据用于模型训练**。同时，您在构建应用或调用模型过程中传输的数据都会经过 AES-256 加密，确保数据安全。"

- 官方文档：https://help.aliyun.com/zh/model-studio/privacy-notice
- 具备 SOC 2、ISO 27001、等保三级等合规认证
- 训练数据披露明确声明："除非客户另行同意，否则不会使用百炼客户的业务数据来开发或改进模型"

**DeepSeek**：

- DeepSeek 声明用户对话内容**默认不会用于模型训练**
- 产品设置中"改进模型体验"开关默认为关闭状态，仅在用户主动开启后，部分对话数据才可能被脱敏后用于分析改进
- 即便开启，数据也会经严格去标识化处理，去除所有个人身份信息
- 本系统通过 API 调用 DeepSeek，不涉及 Web 端"改进体验"开关，API 调用数据不纳入训练
- 隐私政策：https://cdn.deepseek.com/policies/zh-CN/deepseek-privacy-policy.html

**结论**：两家供应商在当前使用方式下，均不会将 API 调用数据用于模型训练。如仍有顾虑，可优先选择阿里云百炼通道（承诺更为明确）。

如后续有更高安全等级需求，可对接企业私有化部署的大模型服务（如本地部署 DeepSeek 开源模型），实现推理数据完全不出内网。

## 开箱即用与多模型支持

### 零配置接入

系统预置公司统一 API Key，用户打开浏览器即可使用，无需自行申请模型服务或配置密钥。仅需添加目标数据库连接信息即可开始查询。

### 多模型灵活切换

内置多模型供应商支持，用户可根据查询场景自行切换：

| 供应商 | 可用模型 | 特点 |
|--------|----------|------|
| DeepSeek | deepseek-v4-flash / deepseek-v4-pro | 推理能力强，适合复杂 SQL 生成 |
| 百炼（DashScope） | qwen3.7-plus / kimi-k2.6 / glm-5.1 等 | 多模型聚合，灵活选择 |

支持自定义 API Key 和 Base URL，团队可根据需要接入其他兼容 OpenAI 协议的模型服务。

### Thinking Mode

支持开启深度推理模式（Thinking Mode），AI 在生成 SQL 前会展示完整的思考过程，便于用户理解查询逻辑、验证推理正确性。
